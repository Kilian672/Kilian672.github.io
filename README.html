Epidemic Model Designer - Development Guide

Installation

  • Make sure to have Visual Studio Code and Node installed (which you have if you are already compiling javascript with VS Code)
  • Start VS Code and open a new terminal: Terminal > New Terminal
  • Enter npm install and wait a few minutes.
  • Open the file package.json, run npm list --depth 0 and make sure that the version numbers coincide with the ones in the development dependencies section.
  • Close the terminal.
  • Use Ctrl+Shift+P to open the search and type Ctrl+Shift+X to open the extension tab and install the following extensions.
    • Debugger for Chrome
    • ESLint
    • Prettier - Code formatter
  • Use Ctrl+Shift+P to open the search and type Preferences: Open Settings (JSON).
  • Replace the contents with the contents of settings.json and save.

Usage

Folder Structure

  • Open this folder in VS code (drag & drop or File > Open Folder).
  • The base folder contains all required configuration files. Only change after alignment for consistency.
  • The folder src contains the javascript source code. The entry script is fixed to index.js.
  • The folder static contains the static website components, in particular the html and the stylesheets.
  • The folder prod contains the files ready for publishing (once generated).

Development

  • Open a new terminal and run npm run dev. This will start a local server (and keep running).
  • Hit F5 to start debugging. Full functionality including automatic reruns, breakpoints and stepping is available.
  • Happy Coding!
  • When the functionality meets your expectations stop debugging, and further the local server in the terminal with Ctrl+C (and confirm).

Build (Stage)

  • For testing and debugging the code after optimisation (bundling, dead code removal and others) run npm run build.
  • Using the local server for debugging is the same as in development. Make sure to test properly, in particular event handlers.

Production

  • After successful testing use npm run prod to produce the output fies.
  • The output files can be found in the folder prod.
